COMMUNICATION_STYLE|

# ── Working With Akien ────────────────────────────────────────────────────────

WORK_ORDER_STYLE|
  1. For non-trivial changes: discuss the plan first, get a "go" before implementing.
  2. For trivial changes (typo, obvious fix, small addition): just do it.
  3. Never make unilateral big architectural decisions. If you're unsure, ask.
  4. When he says "go", implement completely — don't stop mid-way for confirmation.
  5. Report what was done, not what you're about to do.

FILE_EDITING|
  - Pre-approved: proceed on all file edits without asking for confirmation.
  - Always read a file before editing it. Never overwrite blindly.
  - Prefer targeted edits over full rewrites.
  - Don't add comments, docstrings, or type annotations to code you didn't change.
  - Don't add error handling for scenarios that can't happen.

CODE_PHILOSOPHY|
  - Minimum complexity for the current task. No future-proofing for hypotheticals.
  - Three similar lines of code is better than a premature abstraction.
  - If you wouldn't need it this session, don't add it.
  - Avoid backwards-compatibility shims — just change the code.

# ── Documentation Habits ──────────────────────────────────────────────────────

DOCUMENTATION|
  - Decisions and plans go to GitHub issues/discussions — not internal numbering.
  - GitHub is source of truth for all parties (Akien, Claude Code, Igor).
  - Use CSB format for design documents (structured, pipe-delimited, skimmable).
  - DSB (Distilled Semantic Block) = CSB compressed ~60-70% for model ingestion.
  - Gap analysis should live in a design_docs/ file and be kept current.
  - "Written for cold reading" standard: future-you has zero context from this session.

TICKETS|
  - File a ticket for anything not done this session.
  - Close tickets when work is demonstrably complete.
  - Don't use internal #NN shorthand — use GitHub issue numbers.
  - Low-priority tickets are still worth filing; they're a memory, not a commitment.

# ── Source Control ────────────────────────────────────────────────────────────

GIT|
  - Commit at logical checkpoints, not after every edit.
  - Claude Code edits do NOT auto-commit — commit manually.
  - Never --no-verify, never force-push main.
  - Commit messages: "what changed and why" — not just "updated file".
